# Design Doc v2.0: Transductively-Guided Tensorized ILP in XLOG

## 1. Objective

Enable XLOG to dynamically induce logical rules during execution without incurring the massive memory overhead of traditional ILP. We achieve this by treating the rule space as a continuous Adjacency Tensor () gated by a Straight-Through (ST) Gumbel-Softmax binary mask () via DLPack. Furthermore, to accelerate convergence, we integrate a **Transductive Teacher** mechanism that predicts intermediate subgoals to locally constrain the inductive gradient descent.

## 2. Architecture Overview

The ILP system spans the PyTorch neural backbone and four crates in the XLOG workspace:

1. **`pyxlog` / `xlog-neural**`: Handles zero-copy DLPack interchange of the binary rule mask  and backward-pass gradients. Executes the Transductive Teacher.
2. **`xlog-ir` / `xlog-logic**`: Defines the universal `TensorMaskedJoin` node.
3. **`xlog-runtime` / `xlog-cuda**`: Executes the masked join efficiently using native `hash_join_v2` kernels, avoiding the  memory explosion of standard continuous logic.
4. **`xlog-prob`**: Evaluates circuits in log-space using the **Product T-Norm**, proven to be mathematically required for non-vanishing gradient flow to all premises during induction.

### Phase 2: Selective Transductive Guidance (`pyxlog` & PyTorch)

When the inductive system hits a metastability plateau, blind gradient descent over  is too slow. We implement TIIPS-style guidance.

**1. Transductive Subgoal Generation:**
The underlying neural diffusion model (acting as the "Teacher") is prompted to transductively predict the *next intermediate state* of the WMIR without rules (e.g., predicting an intermediate `Velocity` given raw pixels).
**2. Localized Soft Targets:**
This predicted intermediate state is injected into XLOG as a temporary positive example (). The ILP cross-entropy loss is computed directly against this subgoal, drastically narrowing the search space and forcing the  tensor to converge on the rule that satisfies this specific step.

fn execute_tensor_masked_join(&mut self, node: &RirNode) -> Result<CudaBuffer> {
    let RirNode::TensorMaskedJoin { mask_name, left_keys, right_keys, .. } = node else { unreachable!() };
    let mask_tensor = self.neural_registry.get_mask(mask_name)?;
    
    // 1. Extract active rule coordinates (i, j, target_k)
    let active_rules: Vec<(u32, u32, u32)> = self.provider.extract_nonzero_indices(&mask_tensor)?;
    let mut outputs_by_k: HashMap<u32, Vec<CudaBuffer>> = HashMap::new();
    
    // 2. Dispatch native joins only for active rules (solves memory bottleneck)
    for (i, j, k) in active_rules {
        let left_buf = self.store.get_by_id(RelId(i))?;
        let right_buf = self.store.get_by_id(RelId(j))?;
        let joined = self.provider.hash_join_v2(&left_buf, &right_buf, left_keys.clone(), right_keys.clone(), JoinType::Inner)?;
        
        // Tag provenance with mask element M_{i,j,k}
        let prov_joined = self.prob_engine.tag_masked_join(joined, mask_name, i, j, k)?;
        outputs_by_k.entry(k).or_default().push(prov_joined);
    }
    // ... Union results and return
}

### Phase 4: Probabilistic Circuit & Product T-Norm Gradients (`xlog-prob`)

The DeepMind authors proved that Gödel and Łukasiewicz t-norms cause vanishing gradients during induction. XLOG's probability tier solves this inherently.

**1. Log-Space XGCF Circuit (`crates/xlog-prob/src/xgcf.rs`)**
When `tag_masked_join` is called, `xlog-prob` creates an `AND` node:
`AND(f_body1, f_body2, PIR_Lit(M_{i,j,k}))`.
Because XGCF circuits evaluate in log-space, the `AND` node evaluates as . This is the exact mathematical equivalent of the **Product T-Norm** ().

**2. Reverse-Mode Autodiff (`kernels/circuit.cu`)**
During `xgcf_backward_level_*`, the gradients flow evenly to all children of the `AND` node (unlike Gödel's max/min which routes gradients exclusively to the lowest value). The adjoints reaching `PIR_Lit(M_{i,j,k})` are packaged into a DLPack tensor and returned to PyTorch for `W.backward()`.

## 4. Python-Side Orchestration (The DTS-MM Learning Loop)

```python
import pyxlog
import torch
import torch.nn.functional as F

# Initialize XLOG with a learnable Super-Graph template
program = pyxlog.Program.compile("""
    learnable(W_mask) :: rule_template(X, Y) :- body1(X, Z), body2(Z, Y).
""", prob_engine="exact_ddnnf")

N = program.get_schema_size()
W = torch.randn((N, N, N), requires_grad=True, device='cuda')
optimizer = torch.optim.Adam([W], lr=0.1)

def induction_loop(evidence_state, neural_backbone):
    for step in range(100):
        optimizer.zero_grad()
        
        # 1. Transductive Guidance (TIIPS mechanism)
        # Neural model hallucinates the intermediate WMIR state to act as a subgoal
        subgoal_target = neural_backbone.transductive_predict(evidence_state)
        
        # 2. Continuous to Discrete (ST Gumbel-Softmax)
        M_soft = F.gumbel_softmax(W, tau=0.5, hard=False)
        index = M_soft.max(dim=-1, keepdim=True)[1]
        M_hard = torch.zeros_like(W).scatter_(-1, index, 1.0)
        M = (M_hard - M_soft).detach() + M_soft # Straight-through proxy
        
        # 3. Inject mask to XLOG via DLPack
        program.set_rule_mask("W_mask", torch.utils.dlpack.to_dlpack(M))
        
        # 4. GPU Datalog Execution + WMC
        results = program.evaluate(evidence_state) 
        
        # 5. Compute Loss against Transductive Subgoal
        loss = compute_cross_entropy_loss(results, subgoal_target)
        
        # 6. Extract Product T-Norm Gradients from XGCF circuit
        circuit_grads = program.backward(loss) 
        
        # 7. Backprop into PyTorch tensor
        M.backward(circuit_grads)
        optimizer.step()
        
        if loss < CONVERGENCE_THRESHOLD:
            # Harden rule, run Meta-Rule check, and permanently commit
            commit_rule(W)
            break

```

## 5. Deployment Guarantees

1. **Solves the ILP Memory Bottleneck:** Unlike standard differentiable logic which evaluates all rules simultaneously, ST Gumbel-Softmax reduces execution to a single sparse hash-join.
2. **Optimal Gradient Flow:** Leverages XGCF log-space circuits to automatically provide the Product T-Norm gradient distribution proven necessary for ILP induction.
3. **Directed Search Space:** Eliminates "lost" gradient descent by using TIIPS-style transductive subgoals to build targeted loss landscapes.
