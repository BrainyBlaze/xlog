Here is the comprehensive Design Document and Implementation Specification for integrating **Tensorized Differentiable Inductive Logic Programming (ILP)** directly into the XLOG GPU query engine.

This spec focuses strictly on the ILP layer (the PyTorch-XLOG bridge for continuous rule induction) and ignores the broader TNS-MM diffusion backbone.

# Design Doc: Tensorized ILP in XLOG

## 1. Objective

Enable XLOG to dynamically induce logical rules during execution by treating the discrete rule space as a continuous Adjacency Tensor () trained via gradient descent. We will bypass the slow AST-compilation bottleneck by executing a **Tensorized Super-Graph** natively in the GPU, gated by a Straight-Through (ST) Gumbel-Softmax binary mask () provided via DLPack.

## 2. Architecture Overview

The ILP system spans four crates in the XLOG workspace:

1. **`pyxlog` / `xlog-cuda**`: Handles zero-copy DLPack interchange of the binary rule mask  and its backward-pass gradients.
2. **`xlog-ir` / `xlog-logic**`: Defines a universal `TensorMaskedJoin` node representing all possible 2-body joins in the schema.
3. **`xlog-runtime`**: Executes the masked join by dynamically dispatching XLOG's highly-optimized `hash_join_v2` kernels only for coordinates where .
4. **`xlog-prob`**: Tracks provenance of the masked join so that reverse-mode autodiff on the XGCF circuit correctly routes gradients back to the DLPack tensor mask.

### Phase 3: Hot-Loop Execution (`xlog-runtime` & `xlog-cuda`)

The Executor must evaluate the `TensorMaskedJoin` without executing  useless joins.

**1. GPU Non-Zero Extraction (`crates/xlog-cuda/src/provider/mod.rs` & `kernels/pack.cu`)**
Write a small CUDA kernel `extract_nonzero_indices` that scans the  mask array (which is physically small, e.g.,  floats  KB) and outputs a compact list of `(i, j, k)` tuples where .

**2. Dynamic Dispatch (`crates/xlog-runtime/src/executor.rs`)**
Implement the execution logic for the new IR node.

```rust
fn execute_tensor_masked_join(&mut self, node: &RirNode) -> Result<CudaBuffer> {
    let RirNode::TensorMaskedJoin { mask_name, left_keys, right_keys, .. } = node else { unreachable!() };
    
    // 1. Fetch DLPack mask
    let mask_tensor = self.neural_registry.get_mask(mask_name)?;
    
    // 2. Extract active rule coordinates (i, j, target_k)
    // This is a fast GPU->Host async copy of a few bytes
    let active_rules: Vec<(u32, u32, u32)> = self.provider.extract_nonzero_indices(&mask_tensor)?;
    
    let mut outputs_by_k: HashMap<u32, Vec<CudaBuffer>> = HashMap::new();
    
    // 3. Dispatch native joins only for active rules
    for (i, j, k) in active_rules {
        let left_buf = self.store.get_by_id(RelId(i))?;
        let right_buf = self.store.get_by_id(RelId(j))?;
        
        let joined = self.provider.hash_join_v2(
            &left_buf, &right_buf, left_keys.clone(), right_keys.clone(), JoinType::Inner
        )?;
        
        // Tag provenance: This join is gated by mask element M_{i,j,k}
        let prov_joined = self.prob_engine.tag_masked_join(joined, mask_name, i, j, k)?;
        
        outputs_by_k.entry(k).or_default().push(prov_joined);
    }
    
    // 4. Union results into target relation buffers
    for (k, buffers) in outputs_by_k {
        let union_buf = self.provider.union_all(&buffers)?;
        self.store.put(RelId(k), union_buf);
    }
    // Return empty buffer for the node itself, as results are routed via RelId(k)
    Ok(CudaBuffer::empty()) 
}

```

### Phase 4: Probabilistic Circuit & Gradients (`xlog-prob` & `kernels/circuit.cu`)

To enable the continuous tensor  to learn, XLOG must flow gradients back to it.

**1. Provenance Tagging (`crates/xlog-prob/src/provenance.rs`)**
When `tag_masked_join` is called, `xlog-prob` creates a probabilistic fact (`PIR_Lit`) representing the specific mask coordinate .
For every fact  generated by the join of  and , create an `AND` node in the XGCF circuit:
`AND(f_body1, f_body2, PIR_Lit(M_{i,j,k}))`.

**2. Backward Pass (`kernels/circuit.cu` & `crates/xlog-prob/src/exact_gpu.rs`)**
When the loss gradient  is supplied to the root of the XGCF circuit:

1. `xgcf_backward_level_*` kernels propagate the adjoints down the DAG.
2. The adjoints arriving at `PIR_Lit(M_{i,j,k})` represent the gradient .
3. `xlog-prob` aggregates these leaf adjoints into a flat contiguous `TrackedCudaSlice<f32>` matching the shape of the DLPack mask.
4. Python calls `program.get_rule_gradients()`, wraps this slice in a PyTorch tensor, and calls `W.backward(grad_tensor)`.

## 4. Python-Side Orchestration (Example Usage)

Here is how the end-user (or the TNS-MM Control Layer) orchestrates this implementation:

```python
import pyxlog
import torch
import torch.nn.functional as F

# Initialize XLOG
program = pyxlog.Program.compile("""
    learnable(W_mask) :: rule_template(X, Y) :- body1(X, Z), body2(Z, Y).
""", prob_engine="exact_ddnnf")

N = program.get_schema_size()
W = torch.randn((N, N, N), requires_grad=True, device='cuda')
optimizer = torch.optim.Adam([W], lr=0.1)

for step in range(100):
    optimizer.zero_grad()
    
    # 1. Continuous to Discrete (ST Gumbel-Softmax)
    M_soft = F.gumbel_softmax(W, tau=0.5, hard=False)
    index = M_soft.max(dim=-1, keepdim=True)[1]
    M_hard = torch.zeros_like(W).scatter_(-1, index, 1.0)
    M = (M_hard - M_soft).detach() + M_soft # Straight-through trick
    
    # 2. Inject mask to XLOG via DLPack
    program.set_rule_mask("W_mask", torch.utils.dlpack.to_dlpack(M))
    
    # 3. GPU Datalog Execution + WMC
    results = program.evaluate() 
    
    # 4. Compute Loss (e.g., maximizing the probability of observed evidence)
    loss = compute_loss(results)
    
    # 5. Extract Gradients from XGCF circuit
    # Internally maps to root adjoint, runs xgcf_backward_level_*, and returns DLPack
    circuit_grads = program.backward(loss) 
    
    # 6. Backprop into PyTorch tensor
    M.backward(circuit_grads)
    optimizer.step()

```

## 5. Deployment Guarantees

1. **Zero CPU Compilation:** No AST string manipulation or code generation occurs during the learning loop.
2. **Strictly Discrete Execution:** Because  is snapped to exactly `1.0` and `0.0`, the `hash_join_v2` kernels experience no "fuzzy" relational mismatches, preserving Datalog semantics perfectly.
3. **Loop-Invariant Memory:** The mask tensor size is fixed at , guaranteeing stable VRAM allocation without fragmentation during infinite learning loops.
