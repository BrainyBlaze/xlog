### Part 3: PyTorch  XLOG DLPack Integration

The handoff between PyTorch and XLOG must have zero copy overhead to maintain the GPU-resident hot loop.

**1. PyTorch Side (Python/C++):**

```python
import pyxlog
import torch
import torch.nn.functional as F

# W is our N x N x N parameter tensor
def forward_rule_mask(W, tau):
    # Gumbel-Softmax
    M_soft = F.gumbel_softmax(W, tau=tau, hard=False)
    
    # Hard snap (Straight-Through)
    index = M_soft.max(dim=-1, keepdim=True)[1]
    M_hard = torch.zeros_like(W).scatter_(-1, index, 1.0)
    M = (M_hard - M_soft).detach() + M_soft
    
    return M
    
M_tensor = forward_rule_mask(W, tau=0.5)

# Zero-copy export to XLOG
program.set_tensor_mask(torch.utils.dlpack.to_dlpack(M_tensor))

```

**2. The Executor (`crates/xlog-runtime/src/executor.rs`):**
When `execute_node` hits `TensorMaskedJoin`, it reads the mask *before* launching joins.

```rust
fn execute_tensor_masked_join(&mut self, mask_id: RelId, target: RelId) -> Result<CudaBuffer> {
    let mask_buf = self.store.get_dlpack(mask_id)?;
    
    // 1. GPU Kernel: Find non-zero indices (i, j) where M_{i,j,target} == 1.0
    let active_rules = self.provider.find_nonzero_indices(&mask_buf, target)?;
    
    let mut results = Vec::new();
    
    // 2. Only execute joins for the active (i, j) combinations
    for (i, j) in active_rules {
        let left_rel = self.store.get_relation(i)?;
        let right_rel = self.store.get_relation(j)?;
        
        // Dispatch to the native hand-written CUDA hash join
        let joined = self.provider.hash_join_v2(
            &left_rel, &right_rel, 
            vec![1], vec![0], // Standard Z=Z equijoin keys
            JoinType::Inner
        )?;
        results.push(joined);
    }
    
    // 3. Union all active rule derivations
    self.provider.union_all(&results)
}

```
