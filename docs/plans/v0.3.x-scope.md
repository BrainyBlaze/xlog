# Phase 2: v0.3.2 — Language Core

> **Goal:** Complete language foundations for large programs
> **Prerequisites:** v0.3.1 complete

## 2.1 Reversible Symbol Values

**Goal:** Enable symbol-to-string lookup (currently one-way hashed).

| Item | Description |
|------|-------------|
| String intern table | Global intern pool with `u32` → `String` mapping |

**Files:**

- `crates/xlog-logic/src/parser.rs` — intern during parse
- `crates/xlog-cli/src/output.rs` — resolve for display

## 2.2 User-Defined Functions

**Goal:** Allow users to define reusable functions callable from rules.

| Item | Description |
|------|-------------|
| Grammar extension | `func name(args) = expr.` syntax |
| Function registry | Named functions with typed signatures |
| Type inference | Infer return type from expression |
| Inline expansion | Expand function calls during lowering |
| Recursive functions | Support recursion with termination check |
| GPU codegen | Compile function bodies to GPU expressions |

**Example:**
```prolog
func square(X) = X * X.
func distance(X1, Y1, X2, Y2) = pow(square(X2 - X1) + square(Y2 - Y1), 0.5).

close(A, B) :- point(A, X1, Y1), point(B, X2, Y2), distance(X1, Y1, X2, Y2) < 10.0.
```

**Files:**
- `crates/xlog-logic/src/grammar.pest` — function syntax
- `crates/xlog-logic/src/ast.rs` — `FuncDef` AST node
- `crates/xlog-logic/src/function.rs` — new module for function handling
- `crates/xlog-logic/src/lower.rs` — inline expansion

## 2.3 Module System

**Goal:** Organize large programs into reusable modules.

| Item | Description |
|------|-------------|
| Module declaration | `module name.` at file start |
| Import syntax | `use module.` or `use module::{pred1, pred2}.` |
| Qualified names | `module:predicate(...)` references |
| Visibility | Public (default) vs private (`private pred ...`) |
| File resolution | `module foo` → `foo.xlog` in search path |
| Circular import detection | Error on circular dependencies |

**Example:**
```prolog
% File: graph.xlog
module graph.

pred edge(u32, u32).
pred reach(u32, u32).

reach(X, Y) :- edge(X, Y).
reach(X, Z) :- reach(X, Y), edge(Y, Z).
```

```prolog
% File: main.xlog
use graph.

graph:edge(1, 2).
graph:edge(2, 3).

?- graph:reach(1, N).
```

**Files:**
- `crates/xlog-logic/src/grammar.pest` — module/use syntax
- `crates/xlog-logic/src/ast.rs` — `ModuleDecl`, `UseDecl`
- `crates/xlog-logic/src/module.rs` — new module system
- `crates/xlog-logic/src/resolver.rs` — name resolution with modules

# Phase 3: v0.3.3 — Probabilistic Power

> **Goal:** Full-featured probabilistic programming
> **Prerequisites:** v0.3.2 complete

## 3.1 Negation in Exact Inference

**Goal:** Support stratified negation in exact probabilistic inference.

| Item | Description |
|------|-------------|
| Program transformation | Convert negation to positive form via completion |
| Stratum-aware PIR | Build provenance respecting negation strata |
| CNF encoding | Encode negated atoms correctly in Tseitin |
| Test coverage | Negation + evidence + query combinations |

**Current limitation:** Exact inference requires positive-only rule bodies.

**Files:**
- `crates/xlog-prob/src/provenance.rs` — lift positive-only restriction
- `crates/xlog-prob/src/transform.rs` — new negation transformation
- `crates/xlog-prob/src/cnf.rs` — negation encoding

## 3.2 Aggregates in Probabilistic Programs

**Goal:** Support aggregation in probabilistic rules.

| Item | Description |
|------|-------------|
| Aggregate provenance | Track provenance through count/sum/min/max |
| Semiring lifting | Extend WMC semiring for aggregates |
| MC support | Aggregates work in Monte Carlo mode |
| Exact support | Aggregates in exact mode where tractable |

**Example:**
```prolog
0.3::edge(1, 2).
0.5::edge(2, 3).
0.4::edge(1, 3).

degree(X, COUNT(Y)) :- edge(X, Y).
query(degree(1, D)).
```

**Files:**
- `crates/xlog-prob/src/provenance.rs` — aggregate provenance
- `crates/xlog-prob/src/mc.rs` — aggregate evaluation in MC

## 3.3 Alternative Knowledge Compilers

**Goal:** Reduce dependency on D4 with fallback compilers.

| Item | Description |
|------|-------------|
| c2d integration | UCLA c2d compiler as alternative |
| miniC2D integration | Lightweight c2d variant |
| Compiler selection | `--kc-compiler d4|c2d|minic2d` flag |
| Fallback chain | Try D4 → c2d → miniC2D on failure |

**Files:**
- `crates/xlog-prob/src/kc/c2d.rs` — new compiler wrapper
- `crates/xlog-prob/src/kc/minic2d.rs` — new compiler wrapper
- `crates/xlog-prob/src/kc/mod.rs` — compiler selection logic

## 3.4 Importance Sampling

**Goal:** Efficient sampling for rare-event queries.

| Item | Description |
|------|-------------|
| Proposal distribution | Learn proposal from structure |
| Weighted samples | Track importance weights |
| Variance reduction | Stratified/antithetic sampling |
| Diagnostics | Effective sample size reporting |

**Files:**
- `crates/xlog-prob/src/importance.rs` — new module
- `crates/xlog-prob/src/mc.rs` — integrate importance sampling
