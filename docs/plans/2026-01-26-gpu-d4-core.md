# GPU D4 Core (Phase 1) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task.

**Goal:** Implement Phase 1 of the GPU-native D4 compiler: compile a device-resident `GpuCnf` into a device-resident
`GpuXgcf` circuit (XGCF), with deterministic, memory-bounded behavior and zero device->host data-plane transfers.

**Architecture:** Implement a GPU DPLL/D4-style compiler with (1) BFS frontier expansion to a fixed depth and
(2) per-frontier-item block-local DFS to finish compilation. The compiler emits XGCF directly on GPU into a
`GpuCircuitBuilder`, then derives `GpuCircuitLayout` (levels) and constructs `GpuXgcf::from_device(...)`.

**Tech Stack:** Rust (xlog-prob/xlog-cuda), CUDA C++ kernels compiled to PTX via `nvcc` and loaded by `cudarc`.

**Primary spec:** `docs/design/2026-01-22-gpu-native-compilation-design.md` (Sections 2, 2.5, 5.2.4, 6, 8).

## Task 5: Implement per-frontier DFS D4 worker (count + emit)

**Files:**
- Modify: `kernels/d4.cu`
- Modify: `crates/xlog-prob/src/compilation/gpu_d4.rs`

**Behavior:**
- For each frontier item, compile remaining CNF with block-local DFS:
  - unit propagation
  - component decomposition (union-find on variables connected by remaining clauses)
  - branching until solved
  - emit XGCF nodes (Const0/Const1/Lit/And/Or/Decision) with node-level metadata
- Deterministic, memory-bounded:
  - use a two-pass scheme: `d4_compile_count` then prefix-sum then `d4_compile_emit`
  - no global atomics for node allocation

**Tests:**
- compile+verify: for small hand-constructed CNFs (<= 8 vars), run:
  - GPU D4 compile -> `GpuXgcf`
  - GPU equivalence verifier `validate_equivalence_gpu(phi, circuit)` (must UNSAT both ways)

Commit: `feat(d4): compile frontier leaves to XGCF on GPU (count+emit)`
