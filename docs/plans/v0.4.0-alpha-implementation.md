# XLOG v0.4.0-alpha Implementation Plan (Milestone Status)

> This file was originally written as a step-by-step implementation plan.
> As of **2026-02-03**, the majority of the planned functionality exists on `main`.
> This document is now maintained as an **audit + remaining-work checklist** so it stays aligned with the repo.

**Milestone goal:** Prove neural-symbolic integration works end-to-end with the minimal MNIST addition example, and ship a validated, reproducible set of neural examples.

**Architecture (current):**
- Parser supports `nn/4` declarations.
- Python registers PyTorch modules/optimizers and provides tensor sources.
- Neural outputs are imported into the GPU pipeline via DLPack (no host round-trips).
- Exact inference uses GPU compilation + evaluation, with circuit caching for training loops.
- Gradients flow from NLL loss through the compiled circuit back into PyTorch via `Tensor.backward(grad)`.

**Release reality:** `v0.4.0-alpha` is **not tagged**. The latest release tag is `v0.3.2`.

---

## Status Summary (2026-02-03)

- `nn/4` syntax is implemented and tested in Rust parsing tests.
- `xlog-neural` exists (registry, tensor sources, batching/bridge helpers); its Rust tests are feature-gated behind `python-tests`.
- `pyxlog` exposes `Program.compile(...)`, `register_network(...)`, tensor sources, NLL loss helpers, forward/backward, and a simple training loop.
- The minimal neural example exists at `examples/neural/01_minimal/` and is exercised by Python tests.
- Remaining work to claim the milestone is primarily **validation + missing neural examples**, plus a few structural generalizations noted below.

## Task 5: Batched Neural Evaluation (Partially Implemented)

**Plan intent:** group multiple neural predicate calls into batched network forwards.

**Implemented scaffolding:**
- `crates/xlog-neural/src/batch.rs` (`BatchCollector`, `BatchMapping`, `NeuralCall`)
- `python/tests/test_batch_eval.py` (registration/config coverage)

**Current limitation (important):**
- `crates/pyxlog/src/lib.rs` complex-query training path currently evaluates neural calls by iterating indices and calling the
  network module per input (no input stacking). This is correct but not maximally efficient.

**Milestone-level next step (if we want batching to be real, not just config):**
- In `pyxlog`, when `NetworkHandle.batching == true`, gather inputs for a network into a single tensor batch, run one forward,
  then slice outputs back per call.

## Current Constraints / Gaps (Must Be Explicit)

These are the main gaps between the original intent of this plan and what we can currently claim as "done":

- **Release gating (hard):** this repo cannot claim `v0.4.0-alpha` until *all examples* are validated end-to-end and the missing
  neural examples are implemented (see checklist below).
- **Complex-query neural training is template-driven:** the fast-path is optimized around cached circuit templates. Generalizing
  beyond the current supported templates may require more slot mapping + query parsing work.
- **Multi-network complex queries:** the complex-query forward/backward path currently assumes a single registered network; a
  program containing multiple `nn(...)` declarations needs predicate→network resolution at runtime.
- **True batching integration:** batching exists as config + helper scaffolding but is not yet driving a single stacked forward.
