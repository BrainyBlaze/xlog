# v0.4.0-alpha Integrated Validation + Examples + Structural Fixes Design

**Date:** 2026-02-04

## Goal
Ship a production-grade, fully validated neural-symbolic milestone by:
1) building a strict validation harness for *all* examples,
2) implementing the missing neural example suite with real datasets only, and
3) removing structural gaps in the neural pipeline (multi-network resolution + true batching + label mapping correctness).

## Non-Negotiable Constraints
- **Real datasets only.** No synthetic/random fallback anywhere. If data is missing or checksum mismatch, fail fast.
- **CI uses fixed real subsets.** Full datasets are required for release validation.
- **Datasets live per-example** at `examples/neural/<example>/data/` and are gitignored.
- **v0.4.0-alpha is not tagged** until all example validations pass + missing neural examples are implemented.

## Architecture Overview
### 1) Validation Harness
- **Single entrypoint** (script or CLI) that runs all examples in three modes:
  - `ci`: deterministic subsets only (fast, real data)
  - `dev`: subsets + extended smoke checks
  - `release`: full datasets only
- **Dataset manifest** per neural example (checked into repo): `examples/neural/<example>/dataset.json`.
  - source URLs, checksums, split rules, deterministic subset sizes, metric thresholds.
- **Harness responsibilities**:
  - download datasets, verify checksums, enforce subset selection rules
  - execute `train.py --mode {ci|dev|release}` for neural examples
  - run deterministic/probabilistic CLI examples (`scripts/run_xlog_examples.sh`, `examples/prob`, `examples/python`)
  - emit a concise terminal summary + a structured report (append/refresh `docs/VALIDATION_REPORT.md` or dedicated log)

### 2) Missing Neural Example Suite
Required examples (all real datasets):
- **02_coins**: real coin dataset (heads/tails). Logic: win/lose derived from two coin predicates.
- **03_mnist_multidigit**: MultiMNIST/SVHN-style multi-digit recognition; logic composes digits into numbers.
- **04_hwf**: CROHME (handwritten formula) with grammar-constrained logic.
- **05_poker**: card image dataset; logic computes hand ranking.
- **06_clutrr**: CLUTRR relational reasoning; neural encoder produces relation edges, logic answers relation queries.

Each example must include:
- `program.xlog`, `model.py`, `train.py`, `dataset.json`, `README.md`
- `train.py --mode {ci|dev|release}` must enforce dataset presence + checksum validity
- deterministic subset selection in `ci` mode

### 3) Structural Fixes (Neural Pipeline Correctness)
- **Neural predicate registry** created at compile time from parsed `nn/4` declarations:
  - predicate → network name
  - input positions + output variable position
  - exact label list (symbols + integers)
- **Label resolution** must use the declared label list (no heuristics like `a->0`), and fail fast for unknown labels.
- **Multi-network resolution**: complex queries must map each neural predicate call to its correct network, not assume a single network.
- **True batching** in complex-query path:
  - collect neural calls by network
  - stack inputs into a batch tensor per network
  - single forward per network, split outputs by call
  - preserve template ordering to match cached circuit `GpuWeightSlots`

## Data Flow (Complex Query)
1) Parse query → extract neural predicate calls and target label/goal
2) Resolve each call via neural predicate registry
3) Batch calls per network → forward pass (single per network)
4) Convert outputs to device buffers (DLPack) → GPU fast-path
5) GPU returns device-resident loss + gradients → backprop to each network output

## Error Handling
- Missing dataset / checksum mismatch → hard error
- Label not in declared list → hard error
- Network not registered or predicate unresolved → hard error
- CUDA required but unavailable for neural example in `release` mode → hard error

## Testing Strategy
- Add unit tests for label resolution and multi-network query extraction
- Add integration tests for batching path (verify single forward per network)
- Add harness tests (dataset manifest parsing, checksum validation, subset selection)
- CI runs: `ci` mode only with fixed real subsets
- Release run: full dataset validation for all examples

## Success Criteria
- All example validations pass (CLI, prob, python, neural) via harness
- Missing neural examples fully implemented with real datasets
- Neural pipeline supports multi-network programs, correct label mapping, and true batching
- Documentation updated to reflect actual state and validation results
