# v0.4.0-beta Release Design

**Goal:** Tag v0.4.0-beta on current main after fixing the one pre-beta blocker and documenting scope.

**Primary constraint:** Beta = what's on main now. No new features. Term embeddings, extended training controls, and typed query-buffer generalization are deferred to v0.4.0-rc or v0.5.0.

---

## 1. Pre-Beta Blocker: Artifact Config Restore

**Problem:** `LearnedArtifact.load()` drops `config_snapshot` during deserialization (`config_snapshot=None`), even though `save()` persists it. Beta claims reproducible artifacts, so this must be symmetric.

**Approach:**
1. Audit all persisted fields in `LearnedArtifact.save()` vs `load()` for symmetry:
   - `schema_version`, `discovered_rule`, `candidate_map`, `logits`, `soft_probs`, `selected_hard`, `metadata.*`, `config_snapshot`
   - Telemetry exclusion is intentional and documented — keep out of scope.
2. Fix any asymmetries found (expected: `config_snapshot` only).
3. Add roundtrip assertion in `python/tests/test_ilp_artifact.py`: save with known config, load, assert `loaded.config_snapshot == original.config_snapshot`.

**Files:**
- `crates/pyxlog/python/pyxlog/ilp/types.py` (save/load methods)
- `python/tests/test_ilp_artifact.py` (roundtrip test)
