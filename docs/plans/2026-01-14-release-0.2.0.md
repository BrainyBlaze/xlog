# v0.2.0 Release Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Bump project version to `0.2.0`, update release notes, and tag `v0.2.0`.

**Architecture:** Treat versioning as a metadata + release-doc update: update workspace/package versions, regenerate `Cargo.lock`, update `CHANGELOG.md`, then verify and tag.

**Tech Stack:** Rust workspace (Cargo), PyO3 Python package (maturin), Git tags

---

### Task 1: Update version metadata

**Files:**
- Modify: `Cargo.toml` (workspace package version)
- Modify: `crates/xlog-cuda-tests/Cargo.toml`
- Modify: `crates/xlog-gpu-py/pyproject.toml`
- Modify: `Cargo.lock`

**Step 1: Bump workspace version**

Edit `Cargo.toml`:

```toml
[workspace.package]
version = "0.2.0"
```

**Step 2: Bump CUDA test crate version**

Edit `crates/xlog-cuda-tests/Cargo.toml`:

```toml
[package]
version = "0.2.0"
```

**Step 3: Bump Python package version**

Edit `crates/xlog-gpu-py/pyproject.toml`:

```toml
[project]
version = "0.2.0"
```

**Step 4: Regenerate `Cargo.lock`**

Run:

```bash
cargo build -p xlog-core
```

Expected: `Cargo.lock` entries for workspace crates reflect `0.2.0`.

- None

