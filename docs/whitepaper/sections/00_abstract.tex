\begin{abstract}
\noindent
We present xlog, a GPU-native logic programming language that unifies five reasoning paradigms---deterministic Datalog, probabilistic inference, epistemic reasoning over world views, SAT/MaxSAT solving, and differentiable neural-symbolic learning---on a single CUDA runtime, keeping all semantic state device-resident with zero host--device transfers on production paths. Its central contribution is a fully GPU-native knowledge-compilation pipeline (provenance $\to$ CNF $\to$ Decision-DNNF $\to$ arithmetic circuit) in which every compiled circuit is verified, by an on-GPU CDCL solver, to be logically equivalent to its source formula---turning circuit correctness into a machine-checked certificate rather than an assumption. xlog is implemented in Rust with 23 CUDA kernel files and offers zero-copy interoperability with PyTorch, JAX, and Arrow consumers via DLPack and the Arrow C Device interface. Its v0.9.0 release candidate adds epistemic logic programming---\texttt{know}/\texttt{possible} reasoning over world views under founded (FAEEL) and Gelfond-style semantics---executed GPU-natively through a Generate--Propagate--Test plan that reuses the same solver and join machinery. We report ablations on a single development GPU: circuit caching yields a $2.74\times$ end-to-end training speedup (95\% CI $[2.29, 3.18]$) on MNIST addition by amortizing compilation across iterations, and a cost-aware worst-case-optimal join subsystem yields a $27.96\times$ geometric-mean speedup over xlog's own binary-join baseline on skewed program-analysis workloads. These are ablations against xlog's own baselines; we make no head-to-head comparison against external systems, and the contribution is the unified, transfer-free architecture rather than a claim of being fastest.
\end{abstract}
