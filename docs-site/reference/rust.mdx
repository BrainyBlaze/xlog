The xlog Rust workspace publishes the following crates; full API documentation for each is generated by rustdoc and hosted on docs.rs.

| Crate | Responsibility |
|-------|----------------|
| [`xlog-core`](https://docs.rs/xlog-core) | Shared types (`ScalarType`, `Schema`, `AggOp`), traits (`KernelProvider`), errors |
| [`xlog-ir`](https://docs.rs/xlog-ir) | Relational IR nodes (`RirNode`), expressions (`Expr`), execution plans |
| [`xlog-cuda`](https://docs.rs/xlog-cuda) | `CudaKernelProvider`, `GpuMemoryManager`, `CudaBuffer`/`CudaColumn`, PTX embedding, Arrow IPC/C Data + DLPack interop |
| [`xlog-stats`](https://docs.rs/xlog-stats) | `StatsManager` + `StatsSnapshot` (compiler feedback + runtime tracking) |
| [`xlog-runtime`](https://docs.rs/xlog-runtime) | `Executor`, versioned `RelationStore`, profiling, incremental maintenance, adaptive join index cache |
| [`xlog-logic`](https://docs.rs/xlog-logic) | Parser, stratification, lowering (AST → RIR), optimizer (predicate pushdown + join planning), neural predicate syntax (`nn/4`) |
| [`xlog-solve`](https://docs.rs/xlog-solve) | Solver services: GPU CDCL verifier (complete SAT/UNSAT, on-GPU validation) + CLS SAT/MaxSAT (heuristic) |
| [`xlog-gpu`](https://docs.rs/xlog-gpu) | High-level GPU API: deterministic execution + input/output buffers for integration layers |
| [`xlog-prob`](https://docs.rs/xlog-prob) | Probabilistic tier: provenance → CNF → GPU Decision-DNNF compiler → XGCF; exact inference + Monte Carlo sampling + circuit caching |
| [`xlog-cli`](https://docs.rs/xlog-cli) | `xlog` CLI for deterministic and probabilistic execution with Arrow IPC I/O |

<Note>
GPU-dependent items may not fully render on docs.rs: the docs.rs build environment has no CUDA toolkit or GPU, so CUDA-gated modules and feature-dependent APIs can be missing or reduced in the hosted documentation. Building locally with `cargo doc --workspace --no-deps --locked` in a CUDA-enabled environment produces the complete surface.
</Note>

See the [architecture overview](/architecture/overview) for how these crates fit together, and the [CLI Reference](/reference/cli) for the command-line surface built on top of them.
