The docs build generates rustdoc for the full workspace before Mintlify validation and export. Those generated files are copied into `docs-site/generated/rust/` for the exported site and are intentionally not committed.

## Workspace Crates

| Crate | Generated API | Responsibility |
|-------|---------------|----------------|
| `xlog-core` | [rustdoc](/generated/rust/xlog_core/index.html) | Shared types (`ScalarType`, `Schema`, `AggOp`), traits (`KernelProvider`), and common errors. |
| `xlog-ir` | [rustdoc](/generated/rust/xlog_ir/index.html) | Relational IR nodes (`RirNode`), expressions (`Expr`), and execution plans. |
| `xlog-cuda` | [rustdoc](/generated/rust/xlog_cuda/index.html) | CUDA provider, GPU buffers, PTX embedding, Arrow IPC/C Data interop, and DLPack support. |
| `xlog-stats` | [rustdoc](/generated/rust/xlog_stats/index.html) | Runtime and compiler feedback through `StatsManager` and `StatsSnapshot`. |
| `xlog-runtime` | [rustdoc](/generated/rust/xlog_runtime/index.html) | Host-side executor, relation stores, profiling, incremental maintenance, and join index caches. |
| `xlog-logic` | [rustdoc](/generated/rust/xlog_logic/index.html) | Parser, stratification, AST-to-RIR lowering, optimization, and neural predicate syntax. |
| `xlog-solve` | [rustdoc](/generated/rust/xlog_solve/index.html) | Solver services, including GPU CDCL verification and CLS SAT/MaxSAT paths. |
| `xlog-gpu` | [rustdoc](/generated/rust/xlog_gpu/index.html) | High-level GPU execution API and integration buffers. |
| `xlog-prob` | [rustdoc](/generated/rust/xlog_prob/index.html) | Probabilistic provenance, CNF lowering, Decision-DNNF compilation, exact inference, and sampling. |
| `xlog-cli` | [rustdoc](/generated/rust/xlog/index.html) | Command-line execution surface for deterministic and probabilistic runs. The package renders under the Rust crate root `xlog`. |
| `pyxlog` | [rustdoc](/generated/rust/pyxlog/index.html) | Native Python extension crate. It is built for the Python package rather than published to crates.io. |
| `xlog-neural` | [rustdoc](/generated/rust/xlog_neural/index.html) | Neural-symbolic rule learning implementation. It is not a published crates.io package. |
| `xlog-induce` | [rustdoc](/generated/rust/xlog_induce/index.html) | Exact induction implementation. It is not a published crates.io package. |
| `xlog-cuda-tests` | [rustdoc](/generated/rust/xlog_cuda_tests/index.html) | CUDA-facing workspace test crate. It is not a published crates.io package. |
| `xlog-integration` | [rustdoc](/generated/rust/xlog_integration/index.html) | Integration test crate. It is not a published crates.io package. |

## Published Crates

These crates are published on crates.io and also have hosted docs.rs pages:

|-------|---------|
| `xlog-core` | [docs.rs](https://docs.rs/xlog-core) |
| `xlog-ir` | [docs.rs](https://docs.rs/xlog-ir) |

| `xlog-prob` | [docs.rs](https://docs.rs/xlog-prob) |
| `xlog-cli` | [docs.rs](https://docs.rs/xlog-cli) |

<Note>
  docs.rs builds can omit CUDA-gated modules because the hosted build environment has no CUDA toolkit or GPU. The generated site rustdoc is built from the current checkout with `cargo doc --workspace --no-deps --locked`, so it is the reference surface for this documentation site.
</Note>

See the [architecture overview](/architecture/overview) for how these crates fit together, and the [CLI Reference](/reference/cli) for the command-line surface built on top of them.
